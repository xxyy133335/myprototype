## 1\. 项目概述

****江森自控仓储与样机试制管理系统**** 是专为江森自控定制的内部 B 端管理系统，旨在数字化、规范化企业的"仓库管理"核心业务。系统采用现代化 Web 应用架构，使用品牌"江森蓝"与"科技青"作为主视觉，提供高效的物资流转、审批流跟踪及多维数据查询。此 SPEC 仅用作 HTML 原型开发，只开发前端，不开发任何后端。

### 2.2 角色详细说明

-   ****系统管理员 (admin)****：拥有全局最高权限，可查看所有工作台待办、执行所有类型的入库及出库操作，可处理所有审批流程。
-   ****仓管员 (keeper)****：主营仓库物理操作。拥有专属"仓管员入库"权限。在工作台接收并处理由工程师发起的"入库补充审核"流程和"快递发运处理"流程。
-   ****工程师 (engineer)****：主营研发与领料。拥有专属"工程师入库"权限（极简表单，隐去仓库/货位等）。领用出库时，系统自动识别其所属部门及直属经理，用于后续审批。可发起快递发运申请。

## 4\. 核心业务模块与详细字段规格

### 4.1 入库管理 (Inbound)

支持"导出模板"与"批量导入"。页面依角色分为两个 Tab：仓管员入库、工程师入库。

#### 4.1.1 仓管员入库 (单条录入 + 批量导入)

直接物理入库，要求填写详尽的数据，直接写入库存字典。

-   ****表单工具栏****：
    -   表单标题"单条录入"左侧显示。
    -   右侧工具按钮：
        -   `导出模板` 按钮：点击下载 Excel 模板文件，包含所有必填字段的表头。
        -   `批量导入` 按钮：点击唤起文件选择器，支持 `.xlsx, .xls, .csv` 格式文件上传。上传后显示文件名并提示"导入成功"。
-   ****文件上传交互****：
    -   选择文件后，按钮文本变为 `<i class="fas fa-spinner fa-spin"></i> 数据读取中...`
    -   读取完成后变为 `<i class="fas fa-check-circle"></i> {文件名} 导入成功`，按钮边框和文字变为绿色
    -   3 秒后恢复原始状态

-   ****界面可见填写字段 (15个)****：
-   1.  `[必填]` ****物料名称****：文本输入框。
    2.  `[动态必填]` ****物料编号****：文本输入框。仅当物料类别选为“室外机/室内机/压缩机/控制箱”时必填。
    3.  `[必填]` ****项目编号****：文本输入框。
    4.  `[必填]` ****PBU****：下拉选单 (PBU-A, PBU-B, PBU-C)。
    5.  `[必填]` ****免3C****：下拉选单 (是, 否)。
    6.  `[必填]` ****物料类别****：下拉选单 (室外机, 室内机, 压缩机, 控制箱, 工装, 化学品, 新物料, 拆机物料, 工具辅料)。
    7.  `[必填]` ****所属人****：文本输入框。
    8.  `[必填]` ****所属部门****：文本输入框。
    9.  `[必填]` ****PO号****：文本输入框。
    10.  `[选填]` ****供应商编号****：文本输入框。
    11.  `[必填]` ****数量****：数字输入框 (>0)。
    12.  `[必填]` ****单位****：下拉选单 (PCS, KG, 套, 米, 升)。
    13.  `[必填]` ****仓库****：下拉选单 (A, B, C, D, E, F, G, H, I, J, K, 研发中心实验室仓, 原材料一仓)。
    14.  `[必填]` ****货位****：文本输入框 (带 Datalist 快捷下拉，如 A-0-0-0-0)。
    15.  `[必填]` ****单价****：数字输入框 (支持两位小数)。
-   ****系统静默处理字段 (5个)****：  
    16\. `[隐式]` ****申请人****：系统自动获取当前登录账号的真实姓名。  
    17\. `[隐式]` ****入库时间****：系统自动获取提交动作时的时间戳。  
    18\. `[隐式]` ****入库单号****：系统自动生成 (格式: SYS-时间戳)。  
    19\. `[隐式]` ****剩余库存****：首次入库，初始值等于填写的“数量”。  
    20\. `[隐式]` ****账龄周期**** / ****呆滞周期****：初始值系统默认设为“0天”。

#### 4.1.2 工程师入库 (极简录入 + 流转 + 批量导入)

工程师仅填写基础物资信息，提交后生成 `FLOW-xxx` 单号进入待办电子流，由仓管员补充仓储信息后完成正式入库。

-   ****表单工具栏****：
    -   表单标题"单条录入"左侧显示。
    -   右侧工具按钮：
        -   `导出模板` 按钮：点击下载工程师专用的 Excel 模板文件（不含仓库/货位字段）。
        -   `批量导入` 按钮：点击唤起文件选择器，支持 `.xlsx, .xls, .csv` 格式文件上传。

**业务流程说明：**
1. 工程师登录后，入库页面仅显示"工程师入库"Tab，隐藏"仓管员入库"Tab
2. 工程师填写基础物料信息（不含仓库/货位等仓储字段）
3. 点击"提交入库"后，系统生成 `FLOW-时间戳` 格式的流程单号，状态标记为"待仓管审批补充"
4. 记录进入 `pendingWorkflows` 待办队列，推送至仓管员工作台
5. 仓管员在"我的待办"中查看并处理，打开审批弹窗补充仓储信息
6. 仓管员审批通过后，记录正式写入库存字典，流程结束

-   ****界面可见填写字段 (9个)****：
    1.  `[必填]` ****物料名称****：文本输入框， placeholder "请输入物料名称"。
    2.  `[必填]` ****项目编号****：文本输入框， placeholder "请输入关联项目编号"。
    3.  `[必填]` ****PBU****：下拉选单，选项：PBU-A、PBU-B、PBU-C。
    4.  `[必填]` ****免3C****：下拉选单，选项：是 (Yes)、否 (No)。
    5.  `[选填]` ****PO号****：文本输入框带 Datalist 快捷下拉，快捷选项："样品"。
    6.  `[选填]` ****供应商编号****：文本输入框， placeholder "请输入供应商编码 (选填)"。
    7.  `[必填]` ****数量****：数字输入框，placeholder "0"，必须大于0。
    8.  `[必填]` ****单位****：下拉选单，选项：PCS、KG、套、米、升。
    9.  `[必填]` ****单价****：数字输入框，placeholder "0.00"，支持两位小数。

-   ****系统静默处理字段 (5个)****：  
    10\. `[隐式]` ****申请人****：系统自动获取当前登录账号的真实姓名。  
    11\. `[隐式]` ****所属人****：系统自动获取当前登录账号的真实姓名。  
    12\. `[隐式]` ****所属部门****：系统根据工程师登录账号自动绑定所属部门。  
    13\. `[隐式]` ****入库时间****：系统获取提交动作时的时间戳，格式 `YYYY-MM-DD HH:mm`。  
    14\. `[隐式]` ****入库单号****：系统自动生成，格式 `SYS-时间戳后6位`。  
    15\. `[隐式]` ****剩余库存****：首次入库，初始值等于填写的"数量"。  
    16\. `[隐式]` ****账龄周期**** / ****呆滞周期****：初始值系统默认设为"0天"。  
    17\. `[隐式]` ****workflowId****：流程唯一标识，格式 `FLOW-时间戳`。  
    18\. `[隐式]` ****workflowStatus****：流程状态，固定值"待仓管审批补充"。

-   ****仓管员待办弹窗 (Modal) 补充字段****：  
    审批弹窗标题："工程师入库申请 - 仓管补充审核"  
    弹窗信息展示区：物料名称、申请人、数量/单位、项目编号、PO单号、入库单号  
    仓管员需补充字段：
    19\. `[必填]` ****物料类别****：下拉选单 (室外机, 室内机, 压缩机, 控制箱, 工装, 化学品, 新物料, 拆机物料, 工具辅料)。  
    20\. `[动态必填]` ****物料编号****：文本输入框。仅当物料类别选为"室外机/室内机/压缩机/控制箱"时必填。  
    21\. `[必填]` ****仓库****：下拉选单 (A, B, C, D, E, F, G, H, I, J, K, 研发中心实验室仓, 原材料一仓)。  
    22\. `[必填]` ****货位****：文本输入框，带 Datalist 快捷下拉 (如 A-0-0-0-0)。

### 4.2 出库管理 (Outbound)

#### 4.2.1 领用出库 (模糊查询 + 购物车 + 审批流转)

采用**上下全宽**布局，共分两个块：库存检索区（上） → 待领用清单（下）。

-   **库存检索区 (宽表格展示字段)**：  
    展示完整的 20 个字段供比对：`物料名称`、`物料编号`、`申请人`、`项目编号`、`PBU`、`免3C`、`物料类别`、`所属人`、`所属部门`、`PO号`、`供应商编号`、`原入库数量`、`单位`、`入库时间`、`入库单号`、`剩余库存`、`账龄周期`、`呆滞周期`、`仓库`、`货位`。
-   -   `[必填]` **领用数量 (Sticky 操作列)**：数字输入框，前端强校验：不能大于《剩余库存》。
-   -   Enter 键可触发搜索，页面初始化时自动展示全部库存（只展示剩余库存 > 0 的记录）。
-   **待领用清单 (Sticky 表格 · 21列)**：复用上述20字段，将《原入库数量》列替换为蓝色加粗的**领出数量**，并展示《领前余库》。同一入库单号重复添加时合并领用数量，并校验合并后不超出剩余库存上限。右侧图标可移除单条。
-   **领用特有字段 (清单底部表单 · 2列布局)**：
-   -   `[必填]` **领料原因 / 用途**：文本域 (Textarea)，占 2/3 宽。
    -   `[只读不可编]` **审批经理**：占 1/3 宽，系统自动匹配直属经理 (Manager)。
-   **业务逻辑**：提交后生成 `REQ-xxx` 领料电子流，推送至工作台审批。

#### 4.2.2 报废出库 (模糊查询 + 购物车 + 审批流转)

采用**上下全宽**布局，共分两个块：库存检索区（上） → 待报废清单（下）。

-   **库存检索区 (宽表格展示字段)**：  
    展示完整的 20 个字段供比对：`物料名称`、`物料编号`、`申请人`、`项目编号`、`PBU`、`免3C`、`物料类别`、`所属人`、`所属部门`、`PO号`、`供应商编号`、`原入库数量`、`单位`、`入库时间`、`入库单号`、`剩余库存`、`账龄周期`、`呆滞周期`、`仓库`、`货位`。
-   -   `[必填]` **报废数量 (Sticky 操作列)**：数字输入框，右对齐自动粘滔。前端强校验：不能大于《剩余库存》。
-   -   Enter 键可触发搜索，页面初始化时自动展示全部库存。
-   **待报废清单 (Sticky 表格 · 21列)**：复用上述20字段，将《原入库数量》列替换为红色加粗的**报废数量**，并展示《报前余库》。同一入库单号重复添加时合并报废数量，并校验合并后不超出剩余库存上限。右侧 X 图标可移除单条。
-   **报废特有字段 (清单底部表单)**：
-   -   `[必填]` **报废单号**：文本输入框，需填写关联的报废单号。
    -   `[必填]` **上传报废凭证**：文件选择器 (File input)，支持多选 (Multiple)。限定格式：`.jpg, .jpeg, .png, .pdf, .doc, .docx`。
    -   `[只读不可编]` **审批经理**：系统根据当前登录人的账号属性，自动带出其直属经理 (Manager)。
-   **业务逻辑**：提交后生成报废电子流，推送至工作台审批。

### 4.3 移库管理 (Transfer)

**物料处置说明（表格下方提示）：**
1. 所处置的物料范围包括各种已经费用化的零部件、样机、工装及不属于固定资产的低值易耗品（比如小工具）等。

**实现状态：** ✅ 已实现

## 5\. 表单交互与防呆设计 (Global Interactivity)

系统在前端实现了严格的防呆与容错机制，确保流入数据库的数据完整性和有效性：

### 5.1 全局基础交互 (Global Basics)

-   ****必填项视觉引导****：所有必填项标签右侧强制追加红色 `<span class="required">*</span>`。
-   ****防重复提交 (Debounce)****：所有提交类/审批类按钮被点击后，会立刻触发 `disabled = true`，且文本切换为带有旋转 Icon 的“处理中...”或“系统流转中...”，彻底杜绝因网络延迟导致的脏数据或重复发单。
-   ****全局气泡提示 (Toast)****：所有增删改查操作后，无论成功还是失败，均从屏幕上方居中弹出 2.5 秒后自动消失的提示气泡（成功为绿色、失败/阻断为告警红）。

### 5.2 场景化防呆校验 (Scenario-specific Interactivity)

-   ****登录鉴权防呆****：
-   -   账号或密码为空拦截。
    -   验证失败时，输入框发生轻微的****左右震动动画 (Shake)**** 以增强错误感知，并提示红色文字。
    -   监听 `input` 事件，用户重新输入时立刻自动隐藏之前的错误提示。
-   ****动态必填联动与清理****：
-   -   监听【物料类别】下拉框。若选中（室外机/室内机/压缩机/控制箱），则立刻为【物料编号】附加 `required` 属性、显示红星、修改 placeholder 为必填提示。
    -   ****表单重置深度清理****：点击“重置信息”不仅清空数据，还会自动****清除此前提交失败留下的红色边框 (Red Borders)****，并将物料编号的动态状态恢复为初始选填态。
-   ****购物车领用限额校验 (核心业务防呆)****：
-   -   ****数量上限阻断****：用户修改领用数量加入购物车时，系统校验 `输入数量 <= 剩余库存`，否则拦截。
    -   ****合并限额阻断****：若同一物料被二次加入购物车，系统会对数量进行合并，并重新校验 `原购物车数量 + 新增数量 <= 剩余库存`，超出则拦截。
    -   ****空车拦截****：购物车明细为空时，禁用提交动作并提示“请先从左侧检索并添加”。
    -   ****原因追溯拦截****：“领料原因/用途”未填写时，文本域边框变红并拦截提交。
-   ****模态框 (Modal) 审批校验****：
-   -   仓管员在处理流转审批弹窗时，下方的补充字段同样遵循必填项检查。若漏填目标仓库或货位，对应字段变红，流程无法终结。
-   ****异步模拟反馈****：
-   -   导出模板、导入 Excel 时，按钮会有“下载中/读取中”状态，完成后变为“成功”的绿色状态，并延迟 3 秒后重置。
